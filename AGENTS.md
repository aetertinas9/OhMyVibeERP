# ERP 바이브 코딩 실증 프로젝트

"바이브 코딩으로 ERP를 만들 수 있다"를 실증한다. 이 파일은 Claude Code, Codex
등 모든 AI 코딩 에이전트가 공유하는 경량 라우터다. 실제 규칙과 기록은 아래
문서에 있으며, 작업 시작 전 반드시 `docs/workflow.md`를 읽고 따른다.

- `docs/workflow.md` — 필수 작업 규칙(단계별 커밋, 회차별 버그 로그, 충돌 로그,
  지시 기록, 적대적 검증, 정량 지표)
- `Vibe.md` — 사용자 지시 → 실제 구현 내용·줄 수 기록
- `Redteam.md` — 회차별 적대적 검증 기록(분석 전용, 구현에 미반영)
- `Metrics.md` — 회차별 정량 지표(기록 전용)
- `docs/log/` — 회차별 버그 로그 `.log` 산출물(파일명에 브랜치명 포함)
- `docs/conflict-log.md` — 충돌 해결 기록

각 회차는 필수 기록과 단계별 커밋을 모두 마친 뒤 현재 브랜치를 GitHub 원격 저장소에
반드시 push해야 종료된 것으로 본다.
